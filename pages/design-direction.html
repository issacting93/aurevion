AUREVI0N / Design Direction · v0.1

A health app that reads like
instrumentation, not a brochure.

Mono type for data, restraint with color, texture on the things that measure. Treat the user as someone who can read a panel — not someone who needs to be coached through one.

The thread we're pulling on isn't the orange or the dark canvas. It's that every number looks like it was measured, every label looks like a field on a technical drawing, and every chart looks like a calibrated instrument rather than a decoration.

What makes them sophisticated

07 principles
P1

Monospace is structural, not decorative.

Mono carries every number, every label, every timestamp. It signals precision and creates tabular alignment for free. The sans is reserved for the few moments that are actually prose.

P2

The number is the design.

One enormous numeral is the visual hierarchy. No competing illustration, no decorative photograph. The data carries the page; everything else gets out of its way.

P3

One accent, used surgically.

Black, white, and a single orange. No secondary blues, purples, greens stacked together. When green appears it's small and contextual — never claiming equal weight with the brand color.

P4

Bars feel like instruments.

The fill has texture — diagonal hatching, vertical segments, dashed intervals. It reads like an oscilloscope or a fuel gauge, not a generic progress component. The detail is what separates "designed" from "downloaded".

P5

Calibrate it. Show the scale.

0 · 50 · 100 sits above the chart. Tick marks below. The user sees the ruler, not just the reading. This single move converts a bar into a measurement.

P6

Tabular, not typographic.

All numbers align vertically because mono enforces fixed-width digits. Columns of metrics read like a report — disciplined, comparable, scannable.

Black is a primary material.

None of the references fill the canvas. The empty space is structural, not "waste". Confidence to let the page breathe is more sophisticated than any treatment.

03

What changes in the system

3 implications
Type

Geist Mono earns top-billing.

Today: mono is reserved for tiny labels. Tomorrow: every numeric value, label, timestamp, and field name uses mono. Sans is for prose only — coach copy, descriptions, microcopy. This single shift carries 60% of the "sophistication" lift.

AFFECTS · MetricCluster · ChartLabels · Inputs · Calendar · Timeline
Numbers

Numbers go light, not bold.

Hero numbers drop to weight 200–300 and scale up. 1,847 kcal in thin Geist at 100px reads more confidently than the same number bold at 40px. This is counter-intuitive but consistent across every sophisticated reference.

AFFECTS · ProgressRing labels · MetricCluster · Daily summary
Charts

Bars get texture; lines get demoted.

Solid fills become diagonal hatching or segmented columns with a visible scale. Line charts are reserved for one use (long-term trend); everything else uses textured bars or comparison pairs. Status colors get pulled out of charts entirely — the accent and grey carry the comparison.

AFFECTS · ProgressBar · BarMark · LineMark · ComparisonBarChart
Color

Status colors stop competing.

Today the audit shows orange, green, amber, red, blue, purple, cyan all at the same weight. The new system gives the accent 80% of the color budget; success/warning/danger appear only at small sizes (tags, micro-text), never as a large fill. The references' restraint is the lesson.
